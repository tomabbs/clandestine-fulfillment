- Every new read path must follow `docs/system_map/CACHE_ARCHITECTURE.md` (tier class, scope-safe keying, invalidation-first freshness).
- Server-side auth context comes from `requireAuth()` and validated role checks.
- Workspace-scoped admin reads must derive tenant scope on the server from `requireAuth()` rather than trusting a client-cached workspace id for the primary row filter.
- Admin query surfaces must not collapse auth/query failures into the same empty-state copy used for true zero-row results.
- Admin settings pages should avoid hard dependencies on separate bootstrap context reads when the primary domain actions can derive workspace scope from auth directly.

- **SKU matching workspace (2026-04-25):** `/admin/settings/sku-matching` is the staff-only alias-review surface for reconciling canonical warehouse variants to Shopify / WooCommerce / Squarespace product identities without rewriting remote SKUs. - **ShipStation v2 inventory transaction-type semantics (Phase 0 — Patch D2 probe, 2026-04-17):** ShipStation v2 `POST /v2/inventory` enforces a `quantity >= 1` validator on every `transaction_type` that takes a `quantity`/`new_available` field — but the validator is applied asymmetrically across operations and lifecycle stages.
